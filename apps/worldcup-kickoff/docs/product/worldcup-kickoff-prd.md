## 1. プロダクトビジョン

**「ルールも選手も国も知らない人が、開幕日に触って『楽しい・わかった』を最短で体験できる、観て学べる統合ハブ」**

W杯は 4 年に 1 度、普段サッカーを見ない層が一斉に流入する稀有なイベントである。しかし既存アプリ（FIFA 公式 / OneFootball / SofaScore）は統計・速報に最適化されており、「そもそもオフサイドって何」「48 もチームがあってどう勝ち上がるの」「誰を応援すればいいの」という初心者の最初の壁に答えていない。ルール解説はアプリ外の記事・動画に散在し、学びと観戦が分断されている。

worldcup-kickoff は、**学習（ルール）とエンタメ（推し活・予想）を分離せず、大会・試合と直結させた統合ハブ**を提供する。専門用語は出した瞬間にその場で解説され、最初の 3 分でユーザーは「自分の推し国」と「W杯の全体像」を手に入れる。これが唯一の差別化軸である。

### プロダクト原則（意思決定の優先順位）
1. **初心者が開幕日に触って楽しい** を、機能の網羅性より常に優先する。
2. **学びと観戦を分断しない**。ルール・用語は単独ページに閉じず、試合・国・本文中から即アクセスできる。
3. **5 日で開幕に間に合わせる**。間に合わないものは潔く P1/P2 に落とす。
4. **静的でも成立する設計**。ライブスコアがなくても、初心者が学び・推し活・予想を楽しめる体験を作る。

## 2. ターゲットペルソナ

### プライマリ: 「ミーハー初心者・あおいさん（27・会社員）」
- 周囲（職場・恋人・SNS）が W杯で盛り上がっており、自分も乗りたい。
- サッカーのルールはほぼ知らない（オフサイド・PK・カードを説明できない）。
- 有名選手も国も知らない。「日本が出てるらしい」くらいの解像度。
- 通勤・スキマ時間にスマホで触る。長い記事や数字の羅列は読まない。
- 「にわかと思われたくない / でも一から勉強する気力もない」という矛盾を抱える。
- 成功体験 = 「友達との会話で W杯の話についていけた」「応援する国ができた」。

### セカンダリ: 「家族・グループ観戦の幹事」
- 自分はそこそこ見るが、一緒に見る初心者（家族・友人）に説明したい。
- 「これ見せれば分かるよ」と渡せる共有可能な入口を求める。

### 非ターゲット（MVP では狙わない）
- 戦術・スタッツを深掘りするコアサッカーファン（既存アプリで十分満たされている）。
- 賭け・オッズ目的のユーザー。

## 3. 解決する初心者の 3 大課題

| # | 課題 | 初心者の生の声 | 本アプリの解 |
| --- | --- | --- | --- |
| 課題A: ルールがわからない | 「オフサイドって結局なに？なんで今ファウル？」 | ルール図鑑（体験型）＋ 用語じてん（本文ポップオーバー） |
| 課題B: 大会の仕組みがわからない | 「48 チーム？どうやって優勝が決まるの？」 | ブラケット/トーナメント表（48→32→決勝の流れを視覚化）＋ グループ表 |
| 課題C: 誰を応援すればいいかわからない | 「日本以外、応援したい国が思いつかない」 | 推し国診断 → 国図鑑/推し国ページ ＋ 勝敗予想で自分ごと化 |

この 3 課題への回答が P0 の中核を成す。

## 4. 機能一覧とユーザーストーリー

各機能を「初心者として〜したい、なぜなら〜」形式で記述する。

### F1. 開幕カウントダウン付きホーム ＋ オンボーディング導線
- 初心者として、**開幕まであと何日かをひと目で知りたい**。なぜなら、いつ盛り上がるべきかの実感が湧くから。
- 初心者として、**「まず 3 分で W杯を知ろう」という入口から迷わず始めたい**。なぜなら、何から見ればいいか分からないから。
- 初心者として、**ホームから今日の試合・推し国・ルールにすぐ飛べてほしい**。なぜなら、目的が定まっていないから。

### F2. 試合日程 / 今日の試合
- 初心者として、**今日見るべき試合がどれかをホームで知りたい**。なぜなら、104 試合の一覧では何を見ればいいか分からないから。
- 初心者として、**各試合の対戦国・日時（日本時間）・会場を分かりやすく知りたい**。なぜなら、観戦の予定を立てたいから。
- 初心者として、**気になる試合から国の説明にすぐ飛びたい**。なぜなら、知らない国同士だと興味が湧かないから。

### F3. ブラケット / トーナメント表（グループ表含む）
- 初心者として、**48 チームがどう勝ち上がって優勝が決まるのかを 1 枚で理解したい**。なぜなら、文章では仕組みが頭に入らないから。
- 初心者として、**自分の推し国（特に日本）がどの山にいて、次にどこと当たりそうかを知りたい**。なぜなら、応援の見通しが立つから。
- 初心者として、**「3 位でも 8 チーム勝ち上がれる」という今大会特有のルールを優しく説明してほしい**。なぜなら、初耳で混乱するから。

### F4. 推し国診断
- 初心者として、**いくつかの質問に答えるだけで「あなたにおすすめの国」を提案してほしい**。なぜなら、自分で選ぶ基準がないから。
- 初心者として、**診断結果から国図鑑にそのまま進みたい**。なぜなら、結果を見たらもっと知りたくなるから。

### F5. 国図鑑 / 推し国ページ
- 初心者として、**国の基本（場所・愛称・代表カラー・強さの目安）を一目で知りたい**。なぜなら、その国を語れるようになりたいから。
- 初心者として、**「見どころ」「豆知識」「推し選手」を会話のネタとして仕入れたい**。なぜなら、にわかでも盛り上がりたいから。
- 初心者として、**推し国を保存して、次に開いたときホームに出てほしい**。なぜなら、自分の大会にしたいから。

### F6. ルール図鑑（インタラクティブ）
- 初心者として、**オフサイドを「読む」のではなく「動かして体験」したい**。なぜなら、図と操作の方が圧倒的に理解できるから。
- 初心者として、**ポジション・カード・基本ルールを短く区切って学びたい**。なぜなら、一度に大量は無理だから。

### F7. 用語じてん（用語集 + 検索 + 本文ポップオーバー）
- 初心者として、**知らない単語が出たらその場で意味を確認したい**。なぜなら、別アプリで調べると観戦が中断するから。
- 初心者として、**用語を一覧・検索できる辞書がほしい**。なぜなら、後から自分で調べ直したいから。

### F8. 勝敗予想
- 初心者として、**各試合の勝敗を予想して保存したい**。なぜなら、自分が予想すると俄然見たくなるから。
- 初心者として、**自分の的中率を見たい**。なぜなら、ゲーム感覚で大会を追えるから。

## 5. MVP スコープのトリアージ（P0 / P1 / P2）

5 日制約を踏まえ、**「初心者が開幕日に触って 3 大課題が解決し、楽しい」** を満たす最小集合を P0 とする。P0 は絞り、作り込みは P1/P2 に逃がす。

### P0 — 開幕までに必須（これが揃えばローンチ可能）

| 機能 | 初心者にとっての価値（1 行） | スコープの絞り方 |
| --- | --- | --- |
| F1 ホーム（カウントダウン + 3 分導線 + ナビ） | 「何から見ればいいか」の迷いをゼロにする入口 | カウントダウン / 今日の試合 1〜3 件 / 推し国カード / 主要 4 導線のみ |
| F2 試合日程 / 今日の試合 | 104 試合から「今日見るべき試合」だけを差し出す | 日付別リスト + 今日タブ。試合詳細は対戦国・日時(JST)・会場・グループのみ |
| F3 ブラケット/トーナメント表 + グループ表 | 「どう優勝が決まるか」を 1 枚で理解させる | グループ表（12 組）+ 決勝Tの簡易ブラケット。未確定は TBD 表示 |
| F4 推し国診断 | 「誰を応援するか」を 3 分で決めさせる | 5〜7 問・分岐なしのスコアリング。結果 1 国 + 候補 2 国 |
| F5 国図鑑 / 推し国ページ | 推し国を「語れる」ネタを与え、自分の大会にする | 48 国分の最小プロフィール + 見どころ/豆知識/推し選手 各数点 |
| F7 用語じてん（ポップオーバー込み） | 専門用語の壁をその場で取り除く | 30〜50 語。一覧 + 検索 + 本文 `<Term>` ポップオーバー |

> **F6 ルール図鑑・F8 勝敗予想を P0 に入れない判断**: いずれも初心者価値は高いが、F6 の「オフサイド体験」はインタラクション実装コストが重く 5 日では品質リスクが高い。F8 は結果データ（勝敗確定）が静的シードでは入らないため「的中率」が開幕直後に成立しない。代わりに **用語ポップオーバー（F7）でルール理解の最低ラインを担保**し、F6/F8 は P1 で追加する。

### P1 — できれば（開幕後すぐの早期アップデート候補）

| 機能 | 初心者にとっての価値（1 行） |
| --- | --- |
| F6 ルール図鑑（インタラクティブ・オフサイド体験/ポジション/カード） | ルールを「体験」で腹落ちさせ、観戦中の理解を底上げ |
| F8 勝敗予想（グループステージ分の予想保存・参加感） | 予想することで全試合が「自分ごと」になる |
| F1+ オンボーディング「3 分でW杯」フルガイド（複数ステップのウォークスルー） | 初回起動時に最短で全体像と推し国を獲得させる |

> F8 は MVP では「予想を立てて保存・確認できる」段階まで（的中判定なし or 手動結果入力）に留め、結果データ整備後に的中率を有効化する。

### P2 — 将来（大会中・大会後の拡張）

| 機能 | 価値 |
| --- | --- |
| ライブスコア / 速報連携（API 連携・DB 化） | リアルタイム観戦の中核 |
| 勝敗予想の的中率ランキング・共有 / SNS シェア | ソーシャルな盛り上がり |
| 観戦記録・観た試合のログ / 推しメモ | 大会の思い出の蓄積 |
| 多言語対応 / PWA インストール / プッシュ通知 | リテンション・到達拡大 |
| アカウント・クラウド同期（Supabase 導入） | 端末をまたいだ継続利用 |

## 6. 情報設計（IA）とナビゲーション

### 6.1 サイトマップ
```
/                ホーム（統合ハブ・カウントダウン・今日の試合・推し国）
/matches         試合日程（日付別 + 今日タブ）
/matches/[id]    試合詳細（対戦国・日時JST・会場・グループ・両国へのリンク）
/bracket         ブラケット/トーナメント表 + グループ表
/quiz            推し国診断（質問 → 結果）
/countries       国図鑑（48 国一覧・グループ別/地域別フィルタ）
/countries/[code] 国ページ（プロフィール・見どころ・豆知識・推し選手）
/glossary        用語じてん（一覧 + 検索）
（P1）/rules     ルール図鑑
（P1）/predict   勝敗予想
```

### 6.2 グローバルナビゲーション（モバイル: 下部タブ）
P0 の下部タブは 4〜5 に厳選し、認知負荷を抑える。
- **ホーム** / **試合** / **ブラケット** / **国図鑑** / **じてん（用語）**
- 推し国診断（F4）はタブに置かず、ホームの目立つ CTA と国図鑑からの導線で誘導（初回 1 回が中心の機能のため）。

### 6.3 ホームからの主要導線（最重要）
ホームは「迷わせない」ことが全て。以下を縦 1 カラムで上から配置。
1. **開幕カウントダウン**（残り日数を大きく / 開幕後は「大会○日目」表示）
2. **3 分で W杯を知ろう** CTA（オンボーディング / 推し国診断への入口）
3. **今日の試合**（最大 3 件。開幕前は「もうすぐ開幕」状態）
4. **あなたの推し国**（未設定なら診断 CTA、設定済みなら国カード + 次の試合）
5. **W杯のしくみを知る**（ブラケットへ）/ **ルール用語を調べる**（用語じてんへ）

### 6.4 機能間の相互リンク（学びと観戦を分断しない核）
- 試合詳細 → 両対戦国の国ページ
- 国ページ → その国の試合一覧 / 用語ポップオーバー
- ブラケット → 各国ページ / グループ内の試合
- 診断結果 → 推し国ページ（保存導線つき）
- あらゆる本文の専門用語 → `<Term>` ポップオーバーで用語じてんの語義を即表示

## 7. 初心者向け UX 原則（設計・レビューの基準）

1. **専門用語は出した瞬間に即解説**: 本文に専門用語を書いたら、必ず `<Term>` ポップオーバーで語義を添える。未対応の生用語をレビューで検出したら不具合扱い。
2. **最初の 3 分で「楽しい・わかった」を体験させる**: 初回フローは カウントダウン視認 → 診断 → 推し国獲得 → ブラケットで自国の位置確認、を 3 分以内で完了できる導線にする。
3. **数字より物語**: スタッツの羅列を避け、「見どころ」「豆知識」「推し選手」など語れるネタを優先表示。
4. **1 画面 1 メッセージ**: モバイル 1 スクロールで主旨が伝わる粒度に分割。長文・表の横スクロールを避ける。
5. **未確定を正直に・優しく表示**: 抽選未確定の対戦は「対戦相手は抽選後に決定」等、TBD を初心者に分かる言葉で示す（空欄や記号だけにしない）。
6. **にわか肯定のトーン**: 「知らなくて大丈夫」「ここだけ押さえれば会話できる」という安心を与えるコピー。専門家ぶらない。
7. **読まなくても進める**: テキストは補助。タップ・スワイプ・診断など操作で前進できる体験を優先。

## 8. 非機能要件

### 8.1 プラットフォーム / 表示
- **モバイルファースト**: 設計基準幅 375px。タッチターゲット最小 44×44px。下部タブナビ。
- レスポンシブで タブレット / デスクトップでも破綻しない（コンテンツ最大幅を設けセンタリング）。

### 8.2 パフォーマンス
- 静的シードのため初回表示は高速であること（目標: モバイル 4G 相当で LCP < 2.5s）。
- 画像（国旗・選手）は最適化（軽量フォーマット / 遅延読み込み）。国旗は SVG など軽量手段を優先。
- ルート分割により各ページの初期 JS を最小化。

### 8.3 オフライン / データ
- 静的データ（48 国 / 104 試合 / 16 会場 / 用語 / 診断ロジック）はビルド時同梱。**ネット接続なしでも、一度読み込めた静的コンテンツは閲覧可能**であること（ホーム・国図鑑・用語・ブラケットの静的部分）。
- ユーザー入力（推し国・診断結果・予想）は **localStorage** に保存。サーバ送信なし・認証なし。

### 8.4 アクセシビリティ
- 主要操作はキーボード操作可能・フォーカス可視。
- 色だけに依存しない情報伝達（代表カラー表示には国名テキストも併記）。
- 画像に代替テキスト（国旗 = 国名、選手 = 選手名）。
- 文字コントラスト比 WCAG AA 相当を満たす。

### 8.5 プライバシー / 運用
- 個人を特定する情報を収集・送信しない（localStorage はローカル完結）。
- 外部 API 依存なし（MVP）。障害点を最小化し、開幕日のアクセス集中に静的配信で耐える。

## 9. アーキテクチャ方針（実装サブエージェント向けメモ）

- 既定スタック（Next.js App Router + React + TypeScript + Tailwind）を踏襲。**MVP は Supabase / 認証を導入しない**（P2 で検討）。
- 静的データは `src/data/` に型付き TS / JSON で同梱（`teams.ts`, `matches.ts`, `venues.ts`, `groups.ts`, `glossary.ts`, `quiz.ts`）。openfootball 由来データを正とし、未確定は明示的に TBD 値で持つ。
- コンポーネントは **Atomic Design**（atoms ← molecules ← organisms ← templates、逆依存禁止）。
  - 例: atoms = `CountdownDigit`, `FlagIcon`, `Term`(ポップオーバートリガ) / molecules = `MatchCard`, `CountryCard`, `GlossaryItem` / organisms = `TodayMatches`, `GroupTable`, `Bracket`, `QuizFlow` / templates = `HomeTemplate`, `DetailTemplate`。
- 日時は **データを UTC で保持し、表示時に日本時間（JST, UTC+9）へ変換**。コピーには必ず「（日本時間）」を明記。
- localStorage アクセスは単一ユーティリティ層に集約し、無効環境では機能を握りつぶさずグレースフルに縮退（後述 受入条件）。

## 12. 未確定事項 / CPO への確認フラグ
