# キックオフ ⚽️ — 2026 ワールドカップ観戦ガイド

> **ルールも選手も知らなくて大丈夫。** 開幕日に触って3分で「楽しい・わかった」になる、初心者のためのワールドカップ・コンパニオン。

2026 FIFA ワールドカップ（2026/6/11 開幕 〜 7/19 決勝、アメリカ・カナダ・メキシコ共催）を、**普段サッカーを見ない人**が観て学べる日本語・モバイルファーストの統合ハブです。

「オフサイドって何？」「48カ国もあってどう優勝が決まるの？」「日本以外、誰を応援すればいい？」——そんな最初のつまずきにこのアプリ1つで答えます。にわかで大丈夫。ここだけ押さえれば、職場でも家でもワールドカップの話についていけます。

---

## このアプリでできること

初心者にとっての「うれしい」を1行ずつ。

- **開幕カウントダウン付きホーム** — 開幕まであと何日かがひと目でわかり、今やるべきことに迷わない入口。
- **試合日程（全104試合・日本時間表示）** — 104試合の中から「今日見るべき試合」だけを差し出します。
- **トーナメント表／グループ表** — 48カ国がどう勝ち上がって優勝が決まるかを1枚で理解できます。
- **推し国診断** — いくつかの質問に答えるだけで、あなたにぴったりの「応援したい国」が見つかります。
- **国図鑑（出場48カ国＋注目選手）** — その国の見どころ・豆知識・推し選手を、会話のネタとして仕入れられます。
- **ルール図鑑（オフサイド体験つき）** — オフサイドを「読む」のではなく「動かして体験」して腹落ちさせます。
- **用語じてん（46語）** — 知らない言葉が出てきても、その場ですぐ意味を確認できます。
- **勝敗予想** — 勝つ国を予想して保存すれば、ただの試合があなたごとになります。

専門用語は本文中でもその場でポップオーバー解説。学びと観戦を分断しないのが、このアプリの一番のこだわりです。

---

## はじめての体験フロー（おすすめの順番）

最初の3分で「自分の推し国」と「ワールドカップの全体像」が手に入ります。

1. **推し国診断** で質問に答える → あなたに合う応援したい国が決まる
2. **国図鑑** でその国の見どころ・豆知識・推し選手を読む → 語れるネタが増える
3. **試合日程** で推し国の試合がいつあるか（日本時間）をチェック → 観る予定が立つ
4. **勝敗予想** で勝つ国を予想して保存 → 全試合が「自分ごと」になる

迷ったら、まずホームの「推し国診断」から始めてください。

## 画面一覧

| 画面 | パス | 内容 |
| --- | --- | --- |
| ホーム | `/` | 開幕カウントダウン・直近の試合・各機能への入口（統合ハブ） |
| 試合日程 | `/matches` | 全104試合を日付別に一覧（日本時間） |
| トーナメント表／グループ表 | `/bracket` | 12グループ × 4カ国の表と決勝トーナメントの流れ |
| 推し国診断 | `/diagnosis` | 質問に答えて応援する国を見つける |
| 国図鑑 | `/countries` | 出場48カ国の一覧 |
| 国ページ | `/countries/[code]` | 各国のプロフィール・見どころ・豆知識・推し選手 |
| ルール図鑑 | `/rules` | 基本ルール解説（オフサイド体験つき） |
| 用語じてん | `/glossary` | サッカー用語46語の一覧・検索 |
| 勝敗予想 | `/predictions` | 各試合の勝敗を予想して保存 |

> 抽選で未確定の対戦カードは、空欄や記号ではなく「対戦相手は抽選後に決定」のように初心者にわかる言葉で表示します。確定データは再デプロイで差し替えます。

## 技術スタック

| 領域 | 採用技術 |
| --- | --- |
| フレームワーク | Next.js 16（App Router） |
| UI | React 19 / TypeScript / Tailwind CSS v4 |
| アイコン | lucide-react |
| データ整形・検証 | date-fns / zod |
| 保存先 | ブラウザの localStorage のみ |

### アーキテクチャ要約

- **全ページ静的生成（SSG）** — データはビルド時に同梱。サーバー処理やDBへの問い合わせがなく、開幕日のアクセス集中にも静的配信で耐えます。一度読み込めた静的コンテンツはオフラインでも閲覧できます。
- **外部データベース・認証なし** — Supabase などのバックエンドは使いません。障害点を最小化しています。
- **ユーザーデータは localStorage に完結** — 推し国・診断結果・勝敗予想はすべてブラウザ内に保存。サーバーへの送信はなく、個人を特定する情報を収集しません。localStorage が使えない環境でも機能を握りつぶさず、グレースフルに縮退します。
- **日時は UTC で保持し、表示時に日本時間（JST, UTC+9）へ変換** — コピーには必ず「（日本時間）」を明記します。

## データ出典・クレジット

試合日程・対戦カード・会場・グループ分けのデータは、**openfootball プロジェクト**の公開データセットを利用しています。

- **出典**: [openfootball/worldcup.json](https://github.com/openfootball/worldcup.json)（`world-cup.json`）
- **ライセンス**: パブリックドメイン（Creative Commons Zero v1.0 Universal / CC0）。スキーマ・データ・スクリプトはすべてパブリックドメインに供されており、APIキーは不要です。

openfootball コミュニティの公開データへの貢献に感謝します。国名・選手名・各種解説テキストの日本語化、見どころ・豆知識は本アプリで独自に作成・整形したものです。

## ローカル開発

前提: **Node.js >= 22 / pnpm >= 10**（パッケージマネージャーは pnpm に統一。npm / yarn / bun でのインストールはブロックされます）。

```bash
# 依存インストール
pnpm install

# 開発サーバー起動（http://localhost:3000）
pnpm dev

# 本番ビルド（全ページ SSG）
pnpm build

# テスト実行（Vitest）
pnpm test

# Lint
pnpm lint
```

## デプロイ

- **Vercel（Hobby 想定）** にデプロイします。
- Git 連携による自動デプロイを利用します。`main` への push（またはマージ）で本番デプロイ、プルリクエストごとにプレビューデプロイが作成されます。
- 全ページ静的生成のため追加のサーバー設定は不要です。データ確定（抽選結果など）に伴う更新は、データを差し替えて再デプロイすることで反映します。

## ディレクトリ構成（Atomic Design）

UI は Atomic Design（atoms ← molecules ← organisms ← templates、逆方向依存は禁止）で構成しています。

```
apps/worldcup-kickoff/
├── src/
│   ├── app/                      # ルーティング（全ページ SSG）
│   │   ├── page.tsx              # ホーム
│   │   ├── matches/              # 試合日程
│   │   ├── bracket/              # トーナメント表／グループ表
│   │   ├── diagnosis/            # 推し国診断
│   │   ├── countries/            # 国図鑑（[code] で各国ページ）
│   │   ├── rules/                # ルール図鑑（[slug] で各ルール）
│   │   ├── glossary/             # 用語じてん
│   │   └── predictions/          # 勝敗予想
│   ├── components/
│   │   ├── atoms/                # Button, Badge, CountryFlag, Score など最小パーツ
│   │   ├── molecules/            # MatchCard, FeatureTile など機能単位
│   │   ├── organisms/            # HomeHero, GroupTable など独立セクション
│   │   ├── templates/            # HomeTemplate, ListDetailTemplate などレイアウト
│   │   └── providers/            # FavoriteTeamProvider など
│   ├── data/                     # 静的シードデータ（試合・国・選手・用語・ルール）
│   │   └── raw/                  # openfootball 由来の生データ（worldcup-2026.json ほか）
│   └── lib/
│       ├── constants/            # 大会定数（開幕日・グループ・ステージ）
│       ├── data/                 # データ正規化・リポジトリ層
│       ├── diagnosis/            # 推し国診断のロジック
│       ├── domain/               # ドメイン型定義
│       ├── hooks/                # localStorage 連携などのフック
│       ├── storage/              # localStorage 安全アクセス層
│       └── utils/                # 日付（JST 変換）・整形ユーティリティ
└── docs/                         # PRD・マーケティング資料など（日本語）
```
