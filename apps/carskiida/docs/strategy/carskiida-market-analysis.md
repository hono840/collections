# carskiida 市場分析（CSO）

> 作成: 2026-06-07 / 担当: CSO（委譲型PDCA）
> ステータス: ドラフト（承認ゲート1 提出版）

## 判定: 条件付き GO

「小さく深く」始めるなら勝ち筋あり。「カーセンサー級の全網羅 × 全パーツ完全深掘り」を初手で狙うとデータ調達で頓挫するため、その路線は NO-GO。

## 1. 市場分析サマリー

日本のマーケットは以下のプレイヤーで出揃っている:

- **在庫・価格**: カーセンサー / グーネット（掲載53万台規模）
- **口コミ・UGC**: みんカラ（登録107万人・月間UU約1,000万・パーツレビュー約800万件）
- **スペック比較**: 価格.com、greeco channel（57メーカー / 2,289車種 / 10,862グレード）ほか

しかし **「パーツ単位の構造 × 生産工場/生産国 × モデル世代の変遷史」を横断的に束ねた"百科事典"は存在せず、ここに明確な空白がある**。スペックDBの乱立は需要の顕在化を示し、送客先の自動車アフターマーケットは¥20兆超（カスタムは年6%成長）と財布も大きい。

## 2. 競合マップと空白

| カテゴリ | 代表 | 強み | carskiidaが突く穴 |
| --- | --- | --- | --- |
| 在庫・価格 | カーセンサー/グーネット | 在庫網羅・流通 | 車種そのものの構造・歴史は扱わない |
| 口コミ/UGC | みんカラ | パーツレビューの深さ | 構造化・比較・世代史が弱い |
| スペック比較 | 価格.com/greeco | グレード網羅 | 工場/生産国・世代史・設計思想がない |
| 公式 | 各メーカー | 一次情報 | 現行・自社のみ。横断不可 |
| 百科 | Wikipedia | 世代史・網羅 | 非構造・比較不可・欠損まばら |

## 3. 差別化の切り口（勝ち筋）

**「車種を縦軸（世代史）と横軸（パーツ構造）で立体的に引ける、日本語の自動車百科事典」**

1. **世代史ビュー**: 初代→現行を貫き、各世代の型式・パワートレイン・サス形式・安全装備・生産工場移管の変遷を時系列表示
2. **パーツ構造ビュー**: 「この車に何が載るか」をグレード別に構造化した"解剖図"（比較表ではない）
3. **生産地ビュー**: 「どこの工場で作られているか」を横断可視化（vPIC Plant/Country ＋ Wikidata ＋ VIN の WMI で根拠付け）
4. **"なぜ"のエディトリアル ＋ UGC**: 数値だけでなく設計思想・系譜を解説し、エンスージアスト投稿で実車情報を補完

→「みんカラの深さ」を「greecoの網羅構造」で整理し「Wikipediaの世代史」を日本語で引けるようにした第3カテゴリ。初期コアは **エンスージアスト**（深いデータに価値を感じ、UGCで自走拡張）。

## 4. データ調達の実現可能性評価（最大リスク）

- **競合サイトのスクレイピング転載は不可**（利用規約・DB著作権・業務妨害、岡崎市中央図書館事件の前例）。
- **現実解＝ハイブリッド調達**:
  - **NHTSA vPIC**（無料・工場/国/型式/装備を含むが米国仕様）
  - **Wikidata / Wikipedia**（CC、世代・工場の骨格だが欠損まばら）
  - **メーカー諸元の事実値転記**（数値は著作物性が低い）＋ **エンスージアストUGC**
- 各フィールドに **`source`（出典）属性** を持たせる設計が前提。

### 推奨初期スコープ
**人気車種 100〜300 × 直近2〜3世代** に絞れば、合法かつ独自性の高いポジションを取れる。

## 5. マネタイズ仮説（上位3）

1. **専門アフィリエイト/送客（本命）** — パーツ・カスタム（¥20兆超）への高意図トラフィック送客。日本のアフィリエイト市場 2024年度 約5,000億円。
2. **プレミアム（サブスク）/コミュニティ課金** — 高度比較・エクスポート・広告非表示。
3. **データライセンス/B2B（将来）** — 自前で構造化した独自データに限り中古車・保険・メディアへ。

## 6. リスク

- データ調達コスト・欠損（最大）→ スコープ限定 ＋ ハイブリッド調達
- 法務（転載・規約・DB著作権）→ 競合転載ゼロ、Wikipedia継承遵守、画像不使用、事実値のみ転記
- みんカラ/価格.com の本気参入、生成AIによる解説のコモディティ化 → 一次データの構造化と独自UGCを堀に

## 7. ネクストアクション（C-Suite連携）

- **CPO**: 初期スコープとMVP仕様（世代史×パーツ構造ビュー）の要件定義
- **CTO**: vPIC/Wikidata取り込みと「各フィールドにsource属性を持つデータモデル」の技術検証
- **CFO**: 商用VIN/車両DB APIコストとハイブリッド調達（手入力工数含む）の初期試算
- **CMO**: エンスージアスト・コミュニティへの初期アプローチとUGCローンチ戦略
