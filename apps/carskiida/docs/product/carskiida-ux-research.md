# carskiida UXリサーチ（ux-researcher）

> 作成: 2026-06-07 / 担当: ux-researcher（CPO委譲）

## 結論サマリ
carskiida は「在庫・価格を扱わない百科事典」。価格.com/Edmunds/carwow（購買ツール）の「差分ハイライト」「stickyヘッダー」「カテゴリ別グルーピング」のメカニズムは借りるが、画面の頂点に置くのは「世代史×パーツ構造×生産地」という編集された読み物体験。IAの手本は Wikipedia車種ページ（時系列+infobox統一フォーマット）、回遊はみんカラ的コレクション、信頼感は Wikipedia の「要出典」を一級市民として扱う。

## 競合分析

| プロダクト | 借りるべき点 | 避ける/差別化点 |
|-----------|------|------|
| Wikipedia 車種ページ | infobox に製造国・販売期間・駆動・エンジンが統一フォーマットで集約。世代＝「N代目 型式名(年範囲)」で時系列縦展開。出典は上付き番号+末尾分離。青リンク回遊 | 文字中心で見栄え弱い。パーツ構造/生産地が一級表示でなく本文埋没。世代横断比較しづらい |
| Edmunds | 最大4台比較。トリム比較を独立ページ化。"What's new"編集コメント | 購買特化（価格/MPG前提）。米国・英語前提 |
| carwow | 最大3台比較を価格/性能/装備でグループ化。**用語集が1用語1ページ**（馬力=ジムの比喩、PS/kW換算表）。`/guides/glossary/`で体系化 | 比較表の差分強調は公開ページから確認不可。購買主目的 |
| 価格.com | ボディタイプ×メーカーの二大ファセット明快。グレード横断スペック検索。グレード名上部固定 | 在庫/価格が主役。輸入車・旧世代の網羅性・読み物要素が薄い |
| みんカラ | 「自分だけの図鑑にコレクション」。車種→パーツレビュー→関連車種への強い回遊。圧倒的UGC量 | UGC主体で正確性担保が弱い。百科の権威性とは別物 |
| Baymard/NN-g | 「差分のみ表示」トグルが最重要。商品名/サムネをsticky固定。属性はカテゴリ別。モバイル768px以下は横スクロール（見切れ+矢印）or stacked、左列ラベル固定 | （指針）欠損時表示は指針薄く、carskiidaが独自強化 |

## 推奨パターン

### A. 車種詳細ページ（最重要）

**A-1. ヒーロー=「車種アイデンティティ infobox」を一級表示**: 上部固定領域に「世代:全N世代(YYYY–現在)」「生産地:🏭工場名/国旗 国名」「ボディ/駆動/型式系統」を3つの柱として明示。生産地をバッジ化して常時可視＝差別化の核。

**A-2. 世代史=「水平タイムライン(ナビ・sticky)＋縦スクロール(本体)」のハイブリッド**:
```
●───●───●───●───◉───●   ← 世代タイムライン(sticky) クリックでジャンプ
◉ 12代目 E21#型 (2018–現在)
[大判ビジュアル] キャッチコピー/開発コンセプト
┌ この世代のスペック infobox ┐ 生産地/駆動/エンジン/寸法 [出典▸3]
── 主な変更年表 ──
▽ さらに下 → 11代目 …
```
タブでなく縦読み本体（通読の百科体験）+上部sticky目次（特定世代深掘り）で両立。

**A-3. パーツ構造=「解剖図(ホットスポット)＋カテゴリ別アコーディオン」の二刀流**: ビジュアル層（単線の解剖図、クリックで該当カテゴリ）+体系層（カテゴリ別アコーディオン）。初心者は図から、エンスーはアコーディオン展開で。breadth層は解剖図無しでもアコーディオンだけで成立（progressive disclosure）。

### B. 比較体験

**B-1. 2〜4台、カテゴリ別グループ＋「差分のみ表示」トグル必須**: 列ヘッダー（車名+世代+小サムネ）をsticky固定。行はカテゴリ別サブ見出し（寸法/パワートレイン/シャシー/生産地/装備）。「差分のみ表示」トグル（Baymardで最も使われる機能）。色分け+凡例。最大4台（世代横断比較も主用途）。

**B-2. 項目が揃わない時の扱い（最重要差別化）**: 「該当なし(設定なし)」→グレー「—(設定なし)」/「データ未収録」→「ⓘ未収録」+情報提供導線。無知を正直に区別することが権威性（Wikipedia要出典思想）。

### C. 検索・絞り込み・回遊

**C-1. 「メーカー×ボディタイプ」を入口に、carskiida独自に「生産地(国/工場)」「世代の年代」を第3・第4ファセット化**: 価格.comのボディタイプ分類を採用。生産地ファセット（国旗グリッド/工場）と年代/世代ファセット（70年代車、ネオクラ等）を第一級に＝「立体的に引ける」の実装。

**C-2. 関連車回遊は「同世代の兄弟車」「先代/次世代」「同工場で作られた他車」「同ボディタイプ」の4方向**: 「同工場の他車」は他社に無い回遊軸＝生産地テーマの回遊エンジン。

### D. 初心者対応

**D-1. スペック数値に「点線下線ツールチップ」＋「用語集ページ」の二段構え**: 点線下線→タップで1〜2文の簡潔定義（carwowのジム比喩）。末尾「もっと知る▸」→1用語1ページの用語集（`/glossary/{term}`、換算表・関連用語回遊）。

### E. 情報設計（IA）
```
トップ
 ├─ カテゴリ探索: [ボディタイプ][メーカー][生産地/国][年代/世代] ← 4入口
 │   └ 車種一覧（ファセット絞り込み・並べ替え）
 │      └ 車種詳細 ┌ infobox(生産地バッジ) ┐
 │         ├ 世代史(タイムライン+縦スクロール)
 │         │  └ 世代詳細 ├ グレード一覧 → グレード比較(差分トグル)
 │         │             └ パーツ構造(解剖図/アコーディオン) → 各諸元+用語ツールチップ
 │         └ 関連回遊(兄弟車/先代次世代/同工場/同ボディ)
 └─ 厳選ショーケース(深い編集記事・大判ビジュアル) ← 二層構造の"深い層"
```
二層構造の表現: 自動収録層は規格化テンプレで広く均質に。ショーケース層は同じ車種詳細にエディトリアル・オーバーレイ（フルブリード・開発秘話）を重ねる。URLは共通、同一ページ内に編集セクションを増設（回遊を切らない）。

### F. "深さ"の信頼感UX

**F-1. 出典バッジを一級市民に**: 各データブロックに [出典▸N] バッジ。出典の"格"を可視化（公式/二次資料/編集部推定）。「編集部推定」は明示で誠実さ担保。

**F-2. データ欠損の"正直な"見せ方**: 未収録セルは隠さず「ⓘ未収録」+「情報を提供する」導線。ページ単位で「収録充実度メーター」。

## 重要な原則（carskiida固有）
1. **百科事典であって購買ツールではない** — 借りるのは比較表のメカニズムだけ。主役は世代史・パーツ・生産地。
2. **生産地を一級メタ情報に昇格** — infoboxバッジ・ファセット・「同工場回遊」で常時前面に。最大の差別化。
3. **二層ユーザーに段階的開示で同時対応** — インライン/ツールチップで初心者、アコーディオン展開と世代ジャンプで深掘り層。
4. **無知を正直に見せることが信頼になる** — 「設定なし」と「未収録」を区別、出典バッジと充実度メーターで誠実さ可視化。
5. **モバイルファースト** — 768px以下で比較表は横スクロール（見切れ+矢印）or stacked、列ヘッダー/左ラベル固定。世代タイムライン横スワイプ、解剖図タップ拡大。WCAG: ツールチップはタップ/フォーカスでも開く、差分は色のみに頼らずアイコン/ラベル併用。
