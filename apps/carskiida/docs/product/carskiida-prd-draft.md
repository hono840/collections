# carskiida PRD ドラフト（CPO）

> 作成: 2026-06-07 / 担当: CPO（委譲型PDCA）
> ステータス: ドラフト（承認ゲート1 提出版 / Hiroの重要論点回答待ち）

## 1. プロダクトビジョン

カタログでも中古車サイトでもない、車の「中身」を **世代・グレード・搭載パーツ・生産工場** の粒度まで深く横断的に知れる、車好きのための **図鑑**。

- **差別化**: メーカー公式（現行・自社のみ）/ Wikipedia（網羅だが非構造・比較不可）/ カーセンサー（在庫・価格）の隙間。「構造化され比較可能で、世代史まで遡れる深さ」。
- **原則**: ①データの深さと正確性が最優先（出典必須）②「広く浅く」より「狭く深く」から ③比較できることが価値 ④初心者にも開く（用語インライン解説）。

## 2. ターゲットユーザー（ペルソナ）

- **プライマリ**: 買い替え検討者（車選びに役立てたい）
- **セカンダリ**: スペック深掘りマニア（エンスージアスト）
- **ターシャリ**: 用語が分からない初心者

## 3. コア機能リスト（優先度付き）

| 優先 | 機能 | 概要 |
| --- | --- | --- |
| P0 | 車種詳細 | 世代/グレード/パーツ/諸元/生産国・工場を表示 |
| P0 | 検索・絞り込み | メーカー別/ボディタイプ別＋キーワード |
| P0 | 比較表 | 2〜4台、行=項目/列=対象、差分ハイライト |
| P0 | 用語インライン解説 | スペック用語のツールチップ |
| P1 | 関連車回遊 | 同クラス/同世代/競合への導線 |
| P1 | お気に入り収集 | コレクション |
| P2 | 工場の地図ビジュアル / UGC投稿 / ゲーミフィケーション | 拡張 |

**恒久的スコープ外**: 在庫・価格・購入導線（中古車サイト化しない）。

## 4. データモデルの方向性（素案）

```
Manufacturer 1─N CarModel 1─N Generation 1─N Grade
Grade   N─N Part      （同じパーツの複数グレード共有を表現）
Grade   N─N Engine
Generation/Grade ─ Plant ─ ProductionCountry （世代ごとの生産地変遷）
全エンティティに source（出典）を保持
```

→「世代でスペック/パーツが変わる」「同じパーツが横断する」図鑑の深さを構造で担保。

## 5. UX方針（競合調査の結論）

1. 世代タブ/タイムライン切替の詳細ページ
2. パーツのカテゴリ別アコーディオン
3. 列=対象 / 行=項目 ＋ 差分ハイライトの比較表（carwow/Edmunds定番）
4. メーカー別/ボディタイプ別 ＋ キーワード検索（価格.com定番）
5. スペック用語のインラインツールチップ
6. 生産地（工場/国）を一級情報として明示（差別化点）

## 6. MVPスコープ提案

- **合格ライン**: 収録範囲を絞った少数の車種で、図鑑として最低限成立し、車選びに少し役立つ。
- **含む(P0)**: 車種詳細（世代/グレード/パーツ/諸元/生産国・工場）、基本検索・一覧、比較表（最大3〜4対象＋差分ハイライト）、用語インライン解説、データ入力の仕組み。
- **削る**: 高度ファセット/数値検索、工場地図、お気に入り、ゲーミフィケーション、UGC、ネイティブアプリ、全車種網羅。
- **核**: MVPの最大リスクは機能ではなく **データ**。範囲を絞ってでも「深さ」を実証することを優先。

## 7. Hiroに確認すべき重要論点（詳細仕様の前に必須）

1. **【最重要】データの入手・入力方法**: 手入力 / 外部インポート・API・提携 / ハイブリッド。著作権（特に車両画像・ロゴ）・出典・正確性の担保。
2. **【最重要】初期収録範囲**: 国産のみ / 輸入含む。年代範囲（現行のみ / 直近2〜3世代 / 全世代）。
3. **【最重要】MVPの初期収録車種数**: 提案＝国産人気車種10〜30 × 直近1〜2世代から「深さ」実証。
4. **ログイン/アカウントの要否**: ローカル保存で済ますか Supabase Auth を入れるか。
5. **事業性の前提**: 趣味/ポートフォリオ か 事業化か。
6. **プラットフォーム**: Web（モバイルファースト）で良いか、ネイティブ化の優先度。
